When you want a developer blog that stays in Git and still lets you drop in React when you need it, MDX is a comfortable middle ground. You write most of the post in Markdown, then reach for components when you want a callout, a diagram, or an optimized image.

This post is placeholder copy while the blog section is being set up. The important part is that the pipeline works: frontmatter for the index, MDX for the body, and a predictable URL for each slug.

## Example code block

Use fenced blocks with a language tag for syntax highlighting:

```tsx
import { compileMDX } from "next-mdx-remote/rsc";

export default async function Post({ source }: { source: string }) {
  const { content } = await compileMDX({ source });
  return <article className="prose">{content}</article>;
}
```

Inline code like `compileMDX` still uses the inline style.

### Docker

Use `docker` or `dockerfile` as the fence language (both map to Prism’s Docker grammar):

```dockerfile
# syntax=docker/dockerfile:1
FROM node:22-alpine AS deps
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN corepack enable pnpm && pnpm install --frozen-lockfile

FROM deps AS build
COPY . .
RUN pnpm run build

FROM node:22-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=build /app/.next/standalone ./
EXPOSE 3000
CMD ["node", "server.js"]
```

## Tables

Yes. With **GitHub Flavored Markdown** enabled in the MDX pipeline (`remark-gfm`), you can use pipe tables in the usual way: a header row, a separator row with dashes, then body rows.

| Piece        | Role                                      | Where it lives                            |
| ------------ | ----------------------------------------- | ----------------------------------------- |
| Frontmatter  | Title, date, tags for the index           | YAML between `---` at the top of the file |
| MDX body     | Markdown + components where you need them | The rest of the `.mdx` after frontmatter  |
| `compileMDX` | Turns source into React on the server     | `app/(root)/blog/[slug]/page.tsx`         |

Alignment hints work too (`:---`, `:---:`, `---:`) if you want columns left-, center-, or right-aligned.
