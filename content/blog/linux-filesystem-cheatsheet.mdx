Companion cheatsheet for [Linux for Beginners](/blog/linux-for-beginners). Every path below appears in that article (lab examples included where they illustrate typical locations). Use it when you need a quick map of *where things live* on a Linux system.

---

<h2 id="fhs-overview">Filesystem Hierarchy (FHS) — Top Level</h2>

| Path | Purpose | Notes from the course |
| ---- | ------- | --------------------- |
| `/` | Root of the entire filesystem | Often backed by a partition such as `/dev/sda3` |
| `/bin` | Essential user binaries | `cp`, `mv`, `mkdir`, `date`; default shells like `/bin/bash`, `/bin/sh`, `/bin/sync` |
| `/sbin` | Essential system binaries (often root-only) | `/sbin/init` → symlink to `/lib/systemd/systemd` |
| `/boot` | Kernel, initramfs, bootloader files | Early boot sectors; EFI partition often mounted at `/boot/efi` |
| `/dev` | Device files (block & character) | Created/managed by **udev**; e.g. `/dev/sda5`, `/dev/sdb1` |
| `/etc` | System-wide configuration | Service configs, users, network, package repos |
| `/home` | Home directories for regular users | Root’s home is `/root`, not under `/home` |
| `/lib` | Shared libraries for `/bin` and `/sbin` | Includes `/lib/systemd/systemd` and unit files under `/lib/systemd/system/` |
| `/lib64` | 64-bit shared libraries | Runtime dependencies for many programs |
| `/media` | Auto-mount for removable media | Example mount: `/media/usb` |
| `/mnt` | Manual / temporary mount point | Network shares during setup |
| `/opt` | Third-party / optional software | Custom app deployments |
| `/root` | Root user’s home directory | Referenced in `/etc/passwd` entries |
| `/run` | Runtime state (PID files, sockets) | Often `tmpfs`; includes `/run/lock`, `/run/containerd/containerd.sock` |
| `/tmp` | Temporary files | Cron output example: `/tmp/system-report.txt` |
| `/usr` | Userland apps, libraries, docs | Most non-essential binaries and data |
| `/var` | Variable data (logs, mail, spool) | First place to check when troubleshooting |

<h2 id="tree-summary">Directory Tree (Simplified)</h2>

```text
/
├── bin/          # essential user commands
├── boot/         # kernel & bootloader (efi/)
├── dev/          # device nodes (sda*, sdb*, …)
├── etc/          # system configuration
│   ├── apt/sources.list
│   ├── group, os-release, passwd, shadow
│   ├── hosts, resolv.conf, nsswitch.conf
│   ├── network/interfaces
│   ├── sudoers, sudoers.d/
│   ├── systemd/system/   # units & default.target
│   └── yum.repos.d/
├── home/         # user home directories
├── lib/          # libraries; systemd units
│   └── systemd/system/
├── lib64/
├── media/        # auto-mounted removable media
├── mnt/          # manual mounts
├── opt/          # third-party software
├── root/         # root’s home
├── run/          # runtime state (tmpfs)
├── sbin/         # init → systemd
├── tmp/          # temporary files
├── usr/
│   ├── bin/      # most user commands
│   ├── sbin/     # nologin, etc.
│   ├── games/
│   └── share/man/
└── var/
    ├── cache/man/
    ├── log/syslog
    ├── mail/
    ├── spool/
    └── www/
```

> **Tip:** On modern systems, `/bin`, `/sbin`, and `/lib` are often symlinks into `/usr/bin`, `/usr/sbin`, and `/usr/lib`. Layout and merge policies vary by distribution, so always verify on your target host.
