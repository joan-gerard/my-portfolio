I recently worked through KodeKloud's Linux Basics course, and this post is my attempt to document what I learned in a way that's actually useful to return to. If you're getting into DevOps or cloud infrastructure and Linux still feels a bit murky, I hope these notes help.

Linux is everywhere. From the world's fastest supercomputers and most web servers to the majority of smartphones, it underpins modern technology at scale. In the DevOps world specifically, most core tooling was designed with Linux first in mind: Docker started as Linux-only, Ansible controllers run on Linux, and Kubernetes control planes are Linux-based.

That makes Linux not just helpful for this kind of work; it's foundational. The course takes a story-driven approach with hands-on labs, and I've structured my notes to follow the same flow.

### Common challenges

Before diving in, the course acknowledges the typical pain points for Linux beginners, most of which I recognized from my own experience:

- Trouble with navigating Linux CLI
- Navigating directory structures and files
- No experience working with text editors (VI / Nano)
- Different flavors of Linux
- Errors during installation of apps and dependencies (rpm, apt, dpkg, yum)
- Issues with networking between VMs
- Trouble with permissions and security on Linux
- Lack of hands-on practice

### Learning structures

Here's what the course covers, which is also the structure of these notes:

- Interact with the terminal and understand the shell, which serves as the gateway to the Linux kernel
- Study core Linux concepts that help you understand how systems operate
- Explore various package management methods used by different Linux distributions
- Gain confidence in navigating the command line, manipulating files, and using a variety of text editors
- Understand file-level security, configure permissions, and implement essential Linux security practices
- Troubleshoot networking connectivity issues effectively
- Configure storage options including disk formatting, mounting, and creating logical volume groups
- Set up custom applications to start on boot using SYSTEMD, the primary initializer for Linux services
